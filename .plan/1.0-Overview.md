# Freecode — Implementation Plan Overview

## 1.0 Executive Summary

**Project:** Convert opencode (TypeScript/JavaScript monorepo) to Go language, creating "freecode"

**Goal:** A platform-independent AI coding assistant CLI that:
- Runs on FreeBSD 16, Linux, macOS, and IllumOS (OpenSolaris)
- Reads opencode configurations and creates its own
- Integrates oh-my-openagent concepts and configurables
- Provides session tabbing for multiple concurrent sessions
- Binds all services to localhost only (127.0.0.1 and ::1)

**Method:** Incremental conversion preserving all features while leveraging Go idioms.

---

## 2.0 Why Go?

| Factor | TypeScript (opencode) | Go (freecode) |
|--------|----------------------|---------------|
| Binary distribution | Requires Node.js/Bun | Single static binary |
| Cross-compilation | Complex | Built-in `GOOS`/`GOARCH` |
| Performance | JIT overhead | AOT compiled |
| Memory | Higher | Lower footprint |
| Deployment | Multiple files | Single binary |
| Startup | Slow | Fast |

---

## 3.0 Key Design Principles

1. **Feature Parity:** Every opencode feature must exist in freecode
2. **Config Compatibility:** Read opencode config files, generate freecode configs
3. **oh-my-openagent Integration:** All configurables from oh-my-openagent (except: NO PostHog telemetry, NO Discord/Telegram bots, NO automatic update checking)
4. **Platform Independence:** Same code, all platforms
5. **Security First:** All services on 127.0.0.1/::1 only
6. **Session Tabs:** Multiple concurrent sessions in TUI
7. **NO TELEMETRY:** Zero analytics, tracking, or third-party data collection

---

## 8.0 Security Model

All network services bind to:
- `127.0.0.1` (IPv4 loopback)
- `::1` (IPv6 loopback)

No remote access. Services:
- Local API server (port 18792)
- MCP server (port 18793)
- Web UI (port 18791)
