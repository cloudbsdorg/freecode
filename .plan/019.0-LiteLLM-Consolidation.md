# Provider Architecture

## Status: COMPLETED (Phase 1)

## Architecture

Based on opencode analysis, freecode uses a **decorator pattern** where providers wrap base HTTP clients with provider-specific behavior.

### Provider Hierarchy

```
Provider interface (common interface)
│
├── OpenAIProvider ──────► Wraps OpenAI API (NATIVE, PRIMARY)
├── AnthropicProvider ────► Wraps Anthropic API (NATIVE, PRIMARY)
│
├── [OpenAI-Compat] ──────► OpenAI-compatible providers (DECORATORS)
│   ├── MinimaxProvider ──► Wraps OpenAI API (Minimax-specific params)
│   ├── OllamaProvider ───► Wraps OpenAI API (local/cloud Ollama)
│   ├── GroqProvider ──────► Wraps OpenAI API (Groq)
│   ├── PerplexityProvider ► Wraps OpenAI API (Perplexity)
│   └── [etc] ────────────► Any OpenAI-compatible API
│
└── LiteLLMProvider ──────► For LiteLLM proxies (TESTING/MOCKS)
```

### Provider Decorators

Each OpenAI-compatible provider **wraps** the OpenAI provider with:
1. Different base URL
2. Provider-specific model naming
3. Provider-specific request/response handling
4. Provider-specific error handling

**Example: MinimaxProvider**
```go
// MinimaxProvider decorates OpenAIProvider
// Minor differences: max_completion_tokens vs max_tokens, different base URL
type MinimaxProvider struct {
    *OpenAIProvider  // Embeds OpenAI provider
    // Minimax-specific fields if needed
}
```

2. **Static lists** - For providers without /models endpoint (Anthropic)

4. **Config merging** - Combine discovered models with configured models

### OpenAI as Standard

Most providers support OpenAI-compatible API:
- Ollama (via LiteLLM)
- Local models
- Custom gateways

The `Provider` interface uses OpenAI-style request/response:
- Same `Request`/`Response` types
- Same `Generate()` signature
- Provider-specific internal handling

### LiteLLM Role

LiteLLM is for:
- **Testing/Mocking** - httptest.Server mocks in unit tests
- **Providers without native Go SDK** - Only for providers that don't have direct API support
- **Local development** - Testing without API costs

## Context

Freecode currently has separate provider implementations for each AI vendor:
- `internal/provider/openai.go` - Direct OpenAI API calls
- `internal/provider/anthropic.go` - Direct Anthropic API calls

This approach has several issues:
1. **Code duplication** - Both files have nearly identical HTTP boilerplate
2. **Maintenance burden** - API changes require updates to multiple files
3. **Testing complexity** - Each provider needs separate test mocking
4. **Provider lock-in** - Adding new providers requires new code

## Decision

**Use LiteLLM as the unified provider abstraction.**

LiteLLM provides:
- OpenAI-compatible API across 100+ LLMs (OpenAI, Anthropic, Google, Azure, Ollama, Cohere, etc.)
- Local inference via Ollama
- Unified request/response format
- Fallback/failover support
- Streaming support
- Cost tracking

## Implementation

### 1. Create `internal/provider/litellm.go`

```go
package provider

import (
    "bytes"
    "context"
    "encoding/json"
    "fmt"
    "net/http"
    "time"
)

type LiteLLMProvider struct {
    BaseURL string
    APIKey  string
    Client  *http.Client
}

func NewLiteLLMProvider(baseURL, apiKey string) *LiteLLMProvider {
    return &LiteLLMProvider{
        BaseURL: strings.TrimSuffix(baseURL, "/"),
        APIKey:  apiKey,
        Client: &http.Client{
            Timeout: 120 * time.Second,
        },
    }
}

func (p *LiteLLMProvider) Name() string {
    return "litellm"
}

// Generate makes an OpenAI-compatible request through LiteLLM proxy
func (p *LiteLLMProvider) Generate(ctx context.Context, req *Request) (*Response, error) {
    url := p.BaseURL + "/v1/chat/completions"

    payload := map[string]interface{}{
        "model":       req.Model,
        "messages":    req.Messages,
        "temperature": req.Temperature,
        "max_tokens":  req.MaxTokens,
    }

    body, _ := json.Marshal(payload)
    httpReq, err := http.NewRequestWithContext(ctx, "POST", url, bytes.NewReader(body))
    if err != nil {
        return nil, fmt.Errorf("failed to create request: %w", err)
    }

    httpReq.Header.Set("Content-Type", "application/json")
    httpReq.Header.Set("Authorization", "Bearer "+p.APIKey)

    resp, err := p.Client.Do(httpReq)
    if err != nil {
        return nil, fmt.Errorf("request failed: %w", err)
    }
    defer resp.Body.Close()

    var result struct {
        Choices []struct {
            Message struct {
                Content string `json:"content"`
            } `json:"message"`
            FinishReason string `json:"finish_reason"`
        } `json:"choices"`
        Usage struct {
            PromptTokens     int `json:"prompt_tokens"`
            CompletionTokens int `json:"completion_tokens"`
        } `json:"usage"`
    }

    if err := json.NewDecoder(resp.Body).Decode(&result); err != nil {
        return nil, fmt.Errorf("failed to decode response: %w", err)
    }

    if len(result.Choices) == 0 {
        return nil, fmt.Errorf("no choices in response")
    }

    return &Response{
        Content:    result.Choices[0].Message.Content,
        StopReason: result.Choices[0].FinishReason,
        Usage: Usage{
            InputTokens:  result.Usage.PromptTokens,
            OutputTokens: result.Usage.CompletionTokens,
        },
    }, nil
}
```

## Mocking Strategy

### 1. HTTP Mocks (httptest.Server) - PRIMARY APPROACH

Go's `net/http/httptest` provides in-process HTTP server for mock responses. This is the **same pattern already used** in existing `provider_test.go` for OpenAI and Anthropic providers.

**Advantages:**
- No external dependencies
- Fast and deterministic
- Tests run in parallel
- Can simulate error conditions

**Example mock patterns:**

```go
// Mock LiteLLM success response
func TestLiteLLMProviderGenerate(t *testing.T) {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // Verify request
        if r.Header.Get("Authorization") != "Bearer local" {
            t.Errorf("Authorization header = %q, want %q", r.Header.Get("Authorization"), "Bearer local")
        }

        // Simulate LiteLLM response (OpenAI-compatible)
        json.NewEncoder(w).Encode(map[string]interface{}{
            "choices": []map[string]interface{}{{
                "message": map[string]string{"content": "test response"},
                "finish_reason": "stop",
            }},
            "usage": map[string]int{
                "prompt_tokens": 10,
                "completion_tokens": 20,
            },
        })
    }))
    defer ts.Close()

    p := NewLiteLLMProvider(ts.URL, "local")
    resp, err := p.Generate(context.Background(), &Request{
        Model:       "ollama/llama3",
        Messages:    []Message{{Role: "user", Content: "hello"}},
        Temperature: 0.7,
        MaxTokens:   100,
    })

    if err != nil {
        t.Fatalf("Generate() error = %v", err)
    }
    if resp.Content != "test response" {
        t.Errorf("Content = %q, want %q", resp.Content, "test response")
    }
    if resp.StopReason != "stop" {
        t.Errorf("StopReason = %q, want %q", resp.StopReason, "stop")
    }
}

// Mock error response
func TestLiteLLMProviderGenerateError(t *testing.T) {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusInternalServerError)
        json.NewEncoder(w).Encode(map[string]interface{}{
            "error": map[string]string{
                "message": "model not found",
                "type":    "invalid_request_error",
            },
        })
    }))
    defer ts.Close()

    p := NewLiteLLMProvider(ts.URL, "local")

        Model:    "ollama/nonexistent",
        Messages: []Message{{Role: "user", Content: "hello"}},
    })

    if err == nil {
        t.Error("Generate() expected error for HTTP 500")
    }
}

// Mock timeout
```

### 2. Interface-based Mocks

Define a `Provider` interface that can be mocked:

```go
// Provider is the interface for AI model providers
type Provider interface {
    Name() string
    Generate(ctx context.Context, req *Request) (*Response, error)
}

// MockProvider for testing
type MockProvider struct {
    GenerateFunc func(ctx context.Context, req *Request) (*Response, error)
}

func (m *MockProvider) Name() string {
    return "mock"
}

func (m *MockProvider) Generate(ctx context.Context, req *Request) (*Response, error) {
    if m.GenerateFunc != nil {
        return m.GenerateFunc(ctx, req)
    }
    return &Response{Content: "mock response"}, nil
}
```

**Usage in agent tests:**

```go
// Test agent with mock provider
func TestAgentWithMockProvider(t *testing.T) {
    mock := &MockProvider{
        GenerateFunc: func(ctx context.Context, req *Request) (*Response, error) {
            return &Response{
                Content:    "mocked response",
                StopReason: "stop",
            }, nil
        },
    }

    agent := NewAgent(mock)
    resp, err := agent.Run(ctx, "test prompt")
    if err != nil {
        t.Fatalf("Run() error = %v", err)
    }
    if resp.Content != "mocked response" {
        t.Errorf("Content = %q, want %q", resp.Content, "mocked response")
    }
}
```

### 3. Table-Driven Tests

Use table-driven tests for multiple scenarios:

```go
func TestLiteLLMProviderGenerate(t *testing.T) {
    tests := []struct {
        name        string
        model       string
        mockHandler func(w http.ResponseWriter, r *http.Request)
        wantErr     bool
        wantContent string
    }{
        {
            name:  "ollama success",
            model: "ollama/llama3",
            mockHandler: func(w http.ResponseWriter, r *http.Request) {
                json.NewEncoder(w).Encode(map[string]interface{}{
                    "choices": []map[string]interface{}{{
                        "message": map[string]string{"content": "ollama response"},
                    }},
                })
            },
            wantContent: "ollama response",
        },
        {
            name:  "anthropic via litellm",
            model: "anthropic/claude-3-opus",
            mockHandler: func(w http.ResponseWriter, r *http.Request) {
                json.NewEncoder(w).Encode(map[string]interface{}{
                    "choices": []map[string]interface{}{{
                        "message": map[string]string{"content": "claude response"},
                    }},
                })
            },
            wantContent: "claude response",
        },
        {
            name:  "server error",
            model: "ollama/llama3",
            mockHandler: func(w http.ResponseWriter, r *http.Request) {
                w.WriteHeader(http.StatusServiceUnavailable)
            },
            wantErr: true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            ts := httptest.NewServer(http.HandlerFunc(tt.mockHandler))
            defer ts.Close()

            p := NewLiteLLMProvider(ts.URL, "test-key")
            resp, err := p.Generate(context.Background(), &Request{
                Model:    tt.model,
                Messages: []Message{{Role: "user", Content: "hi"}},
            })

            if tt.wantErr {
                if err == nil {
                    t.Error("Generate() expected error")
                }
                return
            }
            if err != nil {
                t.Fatalf("Generate() error = %v", err)
            }
            if resp.Content != tt.wantContent {
                t.Errorf("Content = %q, want %q", resp.Content, tt.wantContent)
            }
        })
    }
}
```

### Mock Summary

| Approach | Use Case | Pros | Cons |
|----------|----------|------|-------|
| `httptest.Server` | Unit tests | Fast, deterministic, no deps | Simulates HTTP only |
| Interface mock | Agent tests | Full control over response | More setup code |
| Table-driven | Multiple scenarios | Clean, organized | More boilerplate |
| Real LiteLLM | Integration tests | Real behavior | Requires infra |

**Recommendation:** Use `httptest.Server` as primary approach (already proven in existing code), interface mocks for agent-level tests, and real LiteLLM+Ollama for integration tests only.

## Ollama Provider (IMPLEMENTED)

### Overview

Ollama provides **local** and **cloud** LLM inference. Supports both:
- **Local**: Running on localhost (no API key)
- **Cloud**: Ollama Cloud (API key required)

**Decorator Pattern:** OllamaProvider wraps OpenAIProvider with:
- Base URL: `http://localhost:11434` (local) or Ollama Cloud
- No auth for local, Bearer token for cloud
- Model names without prefix (e.g., `llama3`, `mistral`)

### Configuration

Environment variables:
- `OLLAMA_BASE_URL` - Base URL (default: `http://localhost:11434`)
- `OLLAMA_API_KEY` - API key for Ollama Cloud (optional)

Config in `config.yaml`:
```yaml
ollama:
  base_url: "http://localhost:11434"
  api_key: ""  # Optional for local, required for cloud

# Or for Ollama Cloud:
ollama:
  base_url: "https://cloud.ollama.ai"
  api_key: "your-api-key"
```

### Model Discovery

Ollama does NOT have a `/v1/models` endpoint. Model list must be fetched via:
```bash
ollama list
```

Or hardcoded common models:
- llama3
- llama3.1
- mistral
- mixtral
- codellama
- neural-chat

### Usage

```bash
# Local Ollama (no API key needed)
export OLLAMA_BASE_URL=http://localhost:11434
freecode run -m ollama/llama3 "Hello"

# Ollama Cloud
export OLLAMA_API_KEY=your-key
export OLLAMA_BASE_URL=https://cloud.ollama.ai
freecode run -m ollama/llama3 "Hello"

# List available models
freecode models --provider ollama
```

### Implementation Pattern

Each OpenAI-compatible provider follows this pattern:

```go
// Provider wraps OpenAIProvider with provider-specific config
type XyzProvider struct {
    *OpenAIProvider  // Embeds base OpenAI client
}

// NewXyzProvider creates provider with Xyz-specific defaults
func NewXyzProvider(apiKey string) *XyzProvider {
    p := &XyzProvider{
        OpenAIProvider: &OpenAIProvider{
            APIKey:  apiKey,
            BaseURL: "https://api.xyz.com/v1",  // Provider-specific
            Client:  &http.Client{Timeout: 60 * time.Second},
        },
    }
    return p
}

func (p *XyzProvider) Name() string {
    return "xyz"
}

// ListModels uses embedded OpenAIProvider's fetch or custom implementation
func (p *XyzProvider) ListModels(ctx context.Context) ([]Model, error) {
    // Some providers use /v1/models, some don't
    return fetchModelsFromURL(ctx, p.BaseURL+"/models", p.APIKey)
}
```

## CLI Models Command (Implemented)

### Usage

```bash
freecode models              # List all available models
freecode models --provider openai  # Filter by provider
freecode models --list       # Simple format (provider/model)
freecode models --refresh    # Force refresh cache
```

### Output Format

```
PROVIDER     MODEL ID                    CONTEXT   CAPABILITIES
--------------------------------------------------------------------------------
anthropic    claude-3-5-haiku            200      tools,vision
anthropic    claude-3-5-opus            200      reasoning,tools,vision
anthropic    claude-3-5-sonnet         200      reasoning,tools,vision
...

# Or simple format
$ freecode models --list
anthropic/claude-3-5-haiku
anthropic/claude-3-5-opus
...
```

### Implementation

Files:
- `internal/provider/models.go` - Model catalog, types, discovery
- `internal/cli/models.go` - CLI command

Key functions:
- `DiscoverFromProvider()` - Fetch models from provider API
- `ModelCatalog` - Thread-safe in-memory cache
- `CatalogService.Save()/Load()` - Persistent cache

### Cache

- Location: `/tmp/freecode-models/catalog.json`
- TTL: 5 minutes
- Auto-loaded on subsequent runs
- Force refresh with `--refresh` flag
