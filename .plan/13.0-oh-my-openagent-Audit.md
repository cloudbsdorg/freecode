## 1. Executive Summary

**Project:** oh-my-openagent
**Repository:** github.com/code-yeongyu/oh-my-openagent
**Type:** OpenCode Plugin/Extension
**Language:** TypeScript (Bun runtime)

This document provides a comprehensive security and technical audit of the oh-my-openagent project. The project presents itself as an OpenCode plugin that enhances the AI coding assistant experience with additional agents, hooks, and integrations.

### Audit Results at a Glance

| Category | Status | Risk Level |
|----------|--------|------------|
| Overall Assessment | **AS ADVERTISED** with caveats | Medium |
| Malicious Code | None detected | Low |
| Data Exfiltration | Potential via hooks | Medium |
| Telemetry | Present, opt-out available | Medium |
| Credential Security | Generally good | Low |
| Permission System | Present, complex | Medium |
| External Network Access | Extensive | Medium-High |

| Claimed Feature | Implemented | Notes |

| 52 Lifecycle Hooks | ✅ YES | Hook system fully implemented |

| Permission System | ✅ YES | ask/allow/deny per tool |

### 2.2 Hidden/Not-Prominently-Disclosed Features

The following are NOT prominently disclosed in documentation:

| Feature | Location | Concern Level |
|---------|----------|---------------|
| **PostHog Telemetry** | `src/shared/posthog.ts` | Medium - sends system data |
| **Discord Integration** | `src/openclaw/reply-listener-discord.ts` | Medium - bot token stored |
| **Telegram Integration** | `src/openclaw/reply-listener-telegram.ts` | Medium - bot token stored |

| **Command Execution Hooks** | `src/shared/command-executor/execute-hook-command.ts` | High - shell command execution |

### 2.3 Verdict

**oh-my-openagent is FUNCTIONALLY as advertised** - it delivers the features it claims. However, it has **UNDISCLOSED capabilities** that could:

1. Send telemetry to third-party analytics services
2. Execute arbitrary commands via hook system
3. Send data to user-configured HTTP endpoints
4. Integrate with Discord/Telegram bots

**Users should be aware these capabilities exist before installing.**

## 3. Security Concerns

### 3.1 Hardcoded PostHog API Key

**File:** `src/shared/posthog.ts:27-28`
**Severity:** Medium (Privacy, not direct security)

```typescript
const DEFAULT_POSTHOG_HOST = "https://us.i.posthog.com"
const DEFAULT_POSTHOG_API_KEY = "phc_CFJhj5HyvA62QPhvyaUCtaq23aUfznnijg5VaaGkNk74"
```

**Issue:** An analytics API key is hardcoded in the source code. While this is a telemetry key (not a secret credential), it means:
- All users send data to the same PostHog project
- The project ID is exposed in any binary
- Usage patterns could be tracked across installations

**Opt-out:** Users can disable via:
- `OMO_DISABLE_POSTHOG=true` environment variable
- `OMO_SEND_ANONYMOUS_TELEMETRY=false` environment variable

### 3.2 HTTP Hooks Can Exfiltrate Data

**File:** `src/hooks/claude-code-hooks/execute-http-hook.ts`
**Severity:** High

The HTTP hook system allows executing arbitrary HTTP requests to external servers:

```typescript
// Only http: and https: schemes allowed (good)
const parsedUrl = new URL(url)
if (!["http:", "https:"].includes(parsedUrl.protocol)) {
  throw new Error("Only http and https protocols are allowed")
}

// Plain HTTP restricted to localhost only (good)
if (parsedUrl.protocol === "http:" && parsedUrl.hostname !== "localhost") {
  throw new Error("Plain HTTP is only allowed for localhost")
}

// Redirects rejected (good)
```

**Risk:** If a user configures a malicious hook URL, tool input data (which could contain code, file contents, or credentials) could be sent to external servers.

### 3.3 Command Execution in Hooks

**File:** `src/shared/command-executor/execute-hook-command.ts:30-34,56-61`
**Severity:** High

```typescript
const expandedCommand = command
  .replace(/^~(?=\/|$)/g, home)
  .replace(/\s~(?=\/)/g, ` ${home}`)
  .replace(/\$CLAUDE_PROJECT_DIR/g, cwd)
  .replace(/\$\{CLAUDE_PROJECT_DIR\}/g, cwd);

// Full shell interpretation
const process = spawn(expandedCommand, [], {
  cwd,
  env: { ...process.env, ...env },
  shell: true,  // DANGEROUS
  timeout: 30000,
})
```

**Issues:**
- Full shell interpretation via `shell: true`
- Inherits all environment variables from parent process
- Command expansion doesn't sanitize special shell characters
- Could be exploited via malicious config to run arbitrary commands

### 3.4 Discord/Telegram Bot Tokens

**File:** `src/openclaw/reply-listener-discord.ts`, `src/openclaw/reply-listener-telegram.ts`
**Severity:** Medium

Both features use bot tokens stored in config:
- Discord: `Bot ${replyListener.discordBotToken}` header
- Telegram: Token embedded in URL

**Concern:** If config files are compromised or shared, bot tokens could be misused. No token rotation mechanism.

## 4. Privacy Concerns

### 4.1 Telemetry Data Collection

**File:** `src/shared/posthog.ts:77-101`
**Severity:** Medium

```typescript
function getSharedProperties(source: PostHogSource): NonNullable<PostHogCaptureEvent["properties"]> {
  const cpus = safeCpus()
  return {
    platform: "oh-my-opencode",
    package_version: packageJson.version,
    runtime: "bun",
    runtime_version: process.versions.bun ?? process.version,
    $os: os.platform(),
    $os_version: os.release(),
    os_arch: os.arch(),
    os_type: os.type(),
    cpu_count: cpus.length,
    cpu_model: cpus.model,           // <-- Potentially identifying
    total_memory_gb: Math.round(os.totalmem() / 1024 / 1024 / 1024),
    locale: Intl.DateTimeFormat().resolvedOptions().locale,
    timezone: Intl.DateTimeFormat().resolvedOptions().timeZone,  // <-- Location hint
    shell: process.env.SHELL,
    ci: Boolean(process.env.CI),
    terminal: process.env.TERM_PROGRAM,
  }
}
```

**Issues:**
- Collects CPU model (could fingerprint specific hardware)
- Collects timezone (approximate location)
- Collects system memory
- No explicit user consent before sending

## 5. Inappropriate Behaviors

### 5.1 File System Operations

**Files:** `src/tools/session-manager/storage.ts`, `src/shared/data-path.ts`

Session storage writes to:
- `~/.local/share/opencode/storage/` (file-based sessions)
- `~/.cache/oh-my-opencode/` (plugin cache)

**Concern:** Session data could contain:
- Conversation history with code
- Tool inputs/outputs (potentially sensitive)
- Credentials passed through agent context

### 5.2 Git Operations

**File:** `src/features/team-mode/team-worktree/manager.ts:19`

```typescript
const process = Bun.spawn({ cmd: ["git", ...args], cwd, stdout: "pipe", stderr: "pipe" })
```

Git worktree operations use the project's git configuration, which may include credentials.

### 5.3 Binary Downloads Without Verification

**Files:** `src/tools/ast-grep/downloader.ts`, `src/tools/grep/downloader.ts`

```typescript
const downloadUrl = `https://github.com/${REPO}/releases/download/${version}/${assetName}`
```

Downloads binaries from GitHub releases without signature verification.

## 11. Summary of Findings

### 11.1 High Priority Concerns

| Issue | Location | Issue Type |

| PostHog API Key Hardcoded | `posthog.ts:27` | Privacy |
| HTTP Hook Exfiltration | `execute-http-hook.ts` | Security |
| Command Execution Hooks | `execute-hook-command.ts` | Security |
| OAuth Token Storage | `storage.ts` | Security |

### 11.2 Medium Priority Concerns

| Issue | Location | Issue Type |
|-------|----------|------------|
| Shell Injection Potential | `execute-hook-command.ts` | Security |
| Discord/Telegram Tokens | `reply-listener-*.ts` | Security |
| Binary Downloads | `downloader.ts` | Security |
| Excessive Telemetry | `posthog.ts` | Privacy |
| NPM Registry Access | `constants.ts` | Privacy |

### 11.3 Lower Priority (Design Trade-offs)

| Issue | Notes |
|-------|-------|
| Permission Complexity | Complex but functional |
| Session Data Sensitivity | Depends on filesystem security |
| Update Checks | Standard practice |

### 11.4 Positive Security Practices

| Practice | Evidence |
|----------|----------|
| No hardcoded API secrets | API keys via env vars |
| Agent tool restrictions | Read-only agents denied writes |
| Permission system | ask/allow/deny support |
| OAuth implementation | Uses PKCE |
| URL validation | Scheme restrictions |
| Subagent isolation | Question denied by default |

### 11.5 Recommendations for Users

1. **Disable telemetry** if privacy is a concern:
   ```bash
   OMO_DISABLE_POSTHOG=true
   OMO_SEND_ANONYMOUS_TELEMETRY=false
   ```

2. **Review hook configurations** before using configs from others

3. **Protect config files** containing Discord/Telegram tokens

4. **Use permission system** - don't set everything to "allow"

5. **Audit agent configurations** - don't give agents more permissions than needed

### 11.6 Recommendations for Project

1. Remove hardcoded PostHog API key (use runtime configuration)
2. Add explicit opt-in for telemetry (currently opt-out)
3. Add GPG signature verification for downloaded binaries
4. Document all network-capable features prominently
5. Consider encrypting stored OAuth tokens
